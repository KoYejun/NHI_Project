# Incident Response Policy

## 목적

Secret 노출 또는 NHI 인증정보 유출 가능성이 확인된 경우 대응 우선순위와 검토 절차를 정의한다.

## 주요 기준

- Critical 위험 항목은 즉시 보안 담당자 검토 대상으로 분류한다.
- High 위험 항목은 빠른 담당자 검토와 접근 로그 확인이 필요하다.
- 사고 분석 리포트에는 Secret 원문을 포함하지 않는다.
- 의심 Secret은 실제 유효성 검증을 외부 서비스에 직접 요청하지 않는다.
- 자동 폐기 또는 권한 회수는 관리자 승인 이후 수행한다.
- 대응 과정과 판단 근거는 감사 가능한 형태로 기록한다.

## 적용 대상

- 운영 환경 Secret 노출
- 인증 헤더가 포함된 로그 노출
- 공개 저장소 내 Token 노출
- 권한 범위가 불명확한 NHI Secret
- 반복 노출된 Secret 후보

## 대응 권고

- 보안 담당자 검토
- 최근 접근 로그 확인
- 영향 범위 분석
- Secret 폐기 및 재발급 검토
- 조치 이력 기록
